## Los 10 Puntos de Evaluacion

Marcar cada punto como:
- **SI** (2 pts) = Cumple completamente
- **PARCIAL** (1 pt) = Existe pero incompleto
- **NO** (0 pts) = No existe

---

### 1. POLITICA DE PRIVACIDAD PUBLICADA

> La ley exige informar a los titulares sobre el tratamiento de sus datos.

**Preguntas a revisar:**
- Tienen politica de privacidad en su sitio web?
- Esta actualizada (menos de 1 ano)?
- Menciona la Ley 21.719 o derechos ARCO-POL?

| [ ] SI - Politica completa y actualizada | 2 |
| [ ] PARCIAL - Existe pero desactualizada | 1 |
| [ ] NO - No existe o muy basica | 0 |

### 2. REGISTRO DE ACTIVIDADES DE TRATAMIENTO

> Obligacion de documentar todas las actividades que involucran datos personales.

**Preguntas a revisar:**
- Tienen un inventario de que datos recopilan?
- Saben donde estan almacenados (sistemas, nubes)?
- Documentan para que usan cada tipo de dato?

| Evaluacion | Puntaje |
|------------|---------|
| [ ] SI - Registro completo y actualizado | 2 |
| [ ] PARCIAL - Saben informalmente pero no documentado | 1 |
| [ ] NO - No tienen idea clara | 0 |

### 3. PROCESO DE CONSENTIMIENTO DOCUMENTADO

> El consentimiento debe ser libre, especifico, informado e inequivoco.

**Preguntas a revisar:**
- Como obtienen el consentimiento de clientes?
- Es especifico por cada finalidad o general?
- Guardan evidencia del consentimiento?

| Evaluacion | Puntaje |
|------------|---------|
| [ ] SI - Proceso formal con evidencia | 2 |
| [ ] PARCIAL - Checkbox pero sin distincion por uso | 1 |
| [ ] NO - Asumen consentimiento implicito | 0 |

**Notas:** _______________________________________________________

---

### 4. MECANISMO DERECHOS ARCO-POL

> Los titulares tienen derecho a Acceder, Rectificar, Cancelar, Oponerse, Portabilidad, Olvido, Limitacion.

**Preguntas a revisar:**
- Tienen forma de recibir solicitudes de clientes?
- Saben como responder a cada tipo de derecho?
- Tienen plazo definido para responder (ley: 10 dias)?

| Evaluacion | Puntaje |
|------------|---------|
| [ ] SI - Proceso formal para todos los derechos | 2 |
| [ ] PARCIAL - Responden ad-hoc cuando llegan | 1 |
| [ ] NO - No han recibido solicitudes / no saben | 0 |

**Notas:** _______________________________________________________

---

### 5. DELEGADO DE PROTECCION DE DATOS (DPO)

> Algunas empresas estan obligadas a designar un DPO segun tamano y tipo de datos.

**Preguntas a revisar:**
- Hay alguien responsable de temas de datos?
- Tiene dedicacion y autoridad para el rol?
- Esta registrado o es informal?

| Evaluacion | Puntaje |
|------------|---------|
| [ ] SI - DPO designado formalmente | 2 |
| [ ] PARCIAL - Alguien lo hace ad-hoc | 1 |
| [ ] NO - Nadie es responsable | 0 |

**Notas:** _______________________________________________________

| [ ] NO - No han evaluado | 0 |

## Proximo Paso Recomendado

### Para Score 0-10 (CRITICO/BAJO)

```
"Tu score de {X}/20 indica brechas significativas. Recomiendo un
diagnostico completo de 4-6 semanas para:

1. Mapear todos tus procesos de datos
2. Crear tu registro de actividades de tratamiento
3. Disenar un roadmap de implementacion priorizado

Inversion: ${rango} USD
Timeline: Empezar en las proximas 2 semanas

Quieres que te prepare una propuesta formal?"
```

### Para Score 11-15 (INTERMEDIO)

```
"Tu score de {X}/20 muestra avances pero hay areas de mejora.
Recomiendo un proyecto enfocado en:

1. {Brecha 1}
2. {Brecha 2}

Esto tomaria {X} semanas y costaria aproximadamente ${rango}.

Te preparo una propuesta?"
```

### Para Score 16-20 (BUENO)

```
"Tu score de {X}/20 es bueno. Felicitaciones por el trabajo hecho.

Sugeriria una auditoria de validacion antes de diciembre 2026 para
asegurar que todo este en orden cuando entre en vigencia.

Si te interesa, me avisas."
```
